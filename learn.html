1. Introduction

C is a powerful general-purpose procedural programming language developed in the early 1970s. It is the foundation for many modern languages like C++, Java, and Python. Learning C gives you a deep understanding of how software really works at the hardware level.

2. Theory

What is C?

A structured, low-level programming language used to develop operating systems, embedded systems, compilers, and applications that require direct hardware manipulation.

History of C

Developed by Dennis Ritchie at Bell Labs in 1972. It was built to rewrite the UNIX operating system. Later standardized by ANSI (C89) and ISO (C99, C11, C17).

Features of C
  • Simple and efficient
  • Fast execution speed
  • Rich set of built-in operators
  • Modular — supports functions
  • Portable across platforms
Applications of C
  • Operating Systems (UNIX, Windows, Linux kernel)
  • Embedded systems & IoT devices
  • Compilers and interpreters
  • Databases (MySQL, PostgreSQL)
  • Game engines and drivers

3. Syntax

The simplest valid C program prints a message to the screen:

#include <stdio.h>

int main() {
    printf("Hello, World!");
    return 0;
}

4. Example

#include <stdio.h>

int main() {
    printf("Welcome to C Programming Hub!\n");
    printf("C is the mother of all modern languages.\n");
    return 0;
}

5. Output

Welcome to C Programming Hub!
C is the mother of all modern languages.

6. Common Mistakes

❌ Forgetting #include <stdio.h> at the top of the file.
❌ Misspelling main as mian or Main.
❌ Missing curly braces { } around the function body.
❌ Not adding a semicolon at the end of statements.

1. Introduction

Every C program follows a specific structure. Understanding each part — header files, the main() function, statements, and the return value — is essential before writing your own programs.

2. Theory

A C program is built from these building blocks:

  • Header Files — Bring in declarations for functions like printf and scanf.
  • main() function — The entry point where execution begins.
  • Statements — Instructions ending with a semicolon ;.
  • return 0; — Signals that the program ended successfully.

3. Syntax

#include <stdio.h>

int main()
{
    printf("Hello World");
    return 0;
}

4. Example (line-by-line explanation)

#include <stdio.h>   // Header file for input/output functions

int main()              // Entry point of every C program
{                       // Start of main function
    printf("Hello World");   // Prints Hello World on screen
    return 0;           // Returns 0 to the OS (success)
}                       // End of main function
#include <stdio.h>

Includes the Standard Input Output header so we can use printf.

int main()

Every C program must have main(). int means it returns an integer.

{ }

Curly braces group multiple statements into one block — the function body.

printf(...)

Function that prints text to the screen. Arguments go inside parentheses.

return 0;

Returns 0 to the operating system — convention for "program ran successfully".

;

Semicolons terminate every statement. Forgetting one is the #1 cause of errors!

5. Output

Hello World

6. Common Mistakes

❌ Missing semicolon after printf("Hello World").
❌ Writing Main() with capital M — C is case-sensitive.
❌ Forgetting to close curly brace } at the end.
❌ Using Printf instead of printf.

1. Introduction

Variables are named containers that store data in memory. Every variable has a type, a name, and a value that can change during program execution.

2. Theory

Definition

A variable is a name given to a memory location that holds a value of a specific type.

Declaration

Variables must be declared before use with the syntax: type name = value;

Rules for Naming
  • Must begin with a letter or underscore (_)
  • Can contain letters, digits, and underscores only
  • Cannot be a reserved keyword (int, return, …)
  • Case-sensitive: Age ≠ age
  • No spaces or special characters (!, @, #)
Naming Conventions
  • Use meaningful names: age, not a
  • Use lowercase with underscores: student_name
  • Avoid single-letter names except in loops
  • Constants use UPPERCASE: MAX_VALUE

3. Syntax

data_type variable_name = value;

Examples of declarations:

int age = 18;
float marks = 89.5;
char grade = 'A';

4. Example

#include <stdio.h>

int main() {
    int age = 20;
    float percentage = 86.5;
    char grade = 'A';

    printf("Age: %d\n", age);
    printf("Percentage: %.2f\n", percentage);
    printf("Grade: %c\n", grade);

    return 0;
}

5. Output

Age: 20
Percentage: 86.50
Grade: A

6. Common Mistakes

❌ int 2age = 20; — variable name can't start with a digit.
❌ char grade = "A"; — use single quotes for chars, not double.
❌ Using a variable before declaring it.
❌ Wrong format specifier: %d for float, %f for int.

1. Introduction

Every variable in C must have a data type. The data type tells the compiler what kind of data the variable will store (whole numbers, decimals, characters, or nothing) and how much memory to reserve. Choosing the right data type makes your program faster and more memory-efficient.

2. Theory

C has 4 primary data types plus void. They are the building blocks for all other types in C.

int
Integer
Size 2 or 4 bytes
Range (typical) -32,768 to 32,767
Format %d

Definition: Stores whole numbers (no decimal point) like 10, -5, 0, 1000.

Syntax: int age = 20;

Usage: Counts, indexes, ages, marks — anywhere a decimal is not needed.

float
Decimal
Size 4 bytes
Precision ~6 decimals
Format %f

Definition: Stores single-precision floating-point numbers (decimals).

Syntax: float price = 19.99f;

Usage: Temperatures, prices, percentages where ~6 digit precision is enough.

double
High Precision
Size 8 bytes
Precision ~15 decimals
Format %lf

Definition: Stores double-precision floating-point numbers (more accurate than float).

Syntax: double pi = 3.14159265358979;

Usage: Scientific calculations, financial apps, anything needing high precision.

char
Character
Size 1 byte
Range -128 to 127
Format %c

Definition: Stores a single character inside single quotes — like 'A', '7', '$'.

Syntax: char grade = 'A';

Usage: Single letters, digits as characters, symbols, yes/no flags.

void
No Value
Size 0 bytes
Range —
Format —

Definition: Means "nothing" or "no value". Used as a function's return type when nothing is returned, or as a generic pointer type.

Syntax: void greet() { printf("Hi"); }

Usage: Functions that don't return a value, generic pointers void *ptr.

3. Syntax — Quick Reference Table

TypeSizeFormat SpecifierExample
int2 / 4 bytes%dint x = 10;
float4 bytes%ffloat x = 3.14f;
double8 bytes%lfdouble x = 3.14159;
char1 byte%cchar x = 'A';
void0 bytes—void func() { }

4. Example

#include <stdio.h>

int main() {
    int age = 20;
    float height = 5.8f;
    double pi = 3.14159265358979;
    char grade = 'A';

    printf("Age: %d\n", age);
    printf("Height: %.1f feet\n", height);
    printf("Pi: %.10lf\n", pi);
    printf("Grade: %c\n", grade);

    return 0;
}

5. Output

Age: 20
Height: 5.8 feet
Pi: 3.1415926536
Grade: A

6. Common Mistakes

❌ Using %d for a float variable — use %f instead.
❌ Writing char grade = "A"; — characters need single quotes, not double.
❌ Forgetting %lf when printing a double with scanf.
❌ Storing large numbers in int when they exceed its range — overflow gives wrong results.
❌ Treating char like a number without understanding ASCII values.

1. Introduction

Operators are symbols that perform operations on variables and values. C is rich in operators — without them, you can't do math, compare values, or build any logic. Mastering operators is essential for writing any real program.

2. Theory

Perform basic mathematical operations on numeric values.

3. Syntax

// Arithmetic
int sum = 10 + 5;       // 15
int rem = 10 % 3;       // 1

// Relational (result is 0 or 1)
int isAdult = (age >= 18);

// Logical
int ok = (marks >= 35) && (attendance >= 75);

// Assignment shorthand
x += 5;   // same as x = x + 5

// Bitwise
int flags = a & b;

4. Example

#include <stdio.h>

int main() {
    int a = 10, b = 3;

    // Arithmetic
    printf("a + b = %d\n", a + b);
    printf("a %% b = %d\n", a % b);

    // Relational
    printf("a > b : %d\n", a > b);

    // Logical
    int result = (a > 5) && (b < 5);
    printf("(a>5) && (b<5) : %d\n", result);

    // Assignment
    a += 5;   // a becomes 15
    printf("a after += 5 : %d\n", a);

    // Bitwise
    printf("5 & 3 = %d\n", 5 & 3);     // 1
    printf("5 << 1 = %d\n", 5 << 1); // 10

    return 0;
}

5. Output

a + b = 13
a % b = 1
a > b : 1
(a>5) && (b<5) : 1
a after += 5 : 15
5 & 3 = 1
5 << 1 = 10

6. Common Mistakes

❌ Confusing = (assignment) with == (comparison). if (x = 5) assigns, if (x == 5) compares.
❌ Dividing two int values and expecting a decimal: 5/2 is 2, not 2.5.
❌ Using % on floats — modulus only works with integers.
❌ Writing && as & or || as | — they are different operators (logical vs bitwise).
❌ Trying to use bitwise operators on float or double — only integers allowed.

1. Introduction

C handles input and output through functions in <stdio.h>. Without I/O, a program is useless — it can't talk to the user. This lesson covers the four most important I/O functions every beginner must master.

2. Theory

printf()
Output

Purpose: Prints formatted text and values to the screen.

Syntax: printf("format", arg1, arg2, ...);

Common format specifiers:

  • %d or %i — integer
  • %f — float / double
  • %c — single character
  • %s — string
  • %lf — double (for scanf)
  • \n — newline
  • \t — tab
scanf()
Input

Purpose: Reads formatted input from the keyboard.

Syntax: scanf("format", &variable);

Important: Variables (except arrays) must be passed with the address-of operator &.

  • scanf("%d", &age); — reads an integer
  • scanf("%lf", &price); — reads a double
  • scanf("%c", &ch); — reads a character
getchar()
Single Char Input

Purpose: Reads one single character from the keyboard (waits for Enter).

Syntax: char ch = getchar();

Usage: Reading one character at a time — useful for menus and y/n prompts.

putchar()
Single Char Output

Purpose: Prints one single character to the screen.

Syntax: putchar(ch);

Usage: Quick character output — equivalent to printf("%c", ch);.

3. Syntax — Quick Comparison

FunctionDirectionHeaderReads / Prints
printfOutput<stdio.h>Formatted text & values
scanfInput<stdio.h>Formatted keyboard input
getcharInput<stdio.h>One character
putcharOutput<stdio.h>One character

4. Example

#include <stdio.h>

int main() {
    int age;
    char initial;

    // printf + scanf
    printf("Enter your age: ");
    scanf("%d", &age);
    printf("You are %d years old.\n", age);

    // getchar + putchar
    printf("Enter your initial: ");
    getchar();             // consume leftover newline
    initial = getchar();
    printf("Your initial is: ");
    putchar(initial);
    putchar('\n');

    return 0;
}

5. Output

Enter your age: 20
You are 20 years old.
Enter your initial: S
Your initial is: S

6. Common Mistakes

❌ Forgetting & in scanf: scanf("%d", age); instead of scanf("%d", &age);.
❌ Using %f for a double in scanf — must use %lf.
❌ Mismatched format specifiers: printing a char with %d, etc.
❌ scanf("%c", &ch); reading the leftover newline from the previous input — call an extra getchar(); to consume it.
❌ Forgetting to #include <stdio.h> — every I/O function needs this header.

7. Practice Questions

  1. Write a program that asks the user's name and age, then prints a greeting like: "Hello Shyam, you are 20 years old."
  2. Read two integers using scanf and print their sum.
  3. Read a character using getchar and print it using both putchar and printf.
  4. Read a float value with scanf("%f", ...) — does it work? What about scanf("%lf", ...)? Why the difference?
  5. Try reading a character right after an integer. Why is the program skipping input? Fix it.

1. Introduction

Decision-making statements let your program choose between different paths based on conditions. Without them, every program would only run top-to-bottom. These are the building blocks of all real-world logic.

2. Theory & Syntax

if statement
Single Choice

Theory: Runs a block of code only if the condition is true. If false, the block is skipped.

Syntax:

if (condition) {
    // runs only when condition is true
}
if-else
Two Choices

Theory: One block runs if the condition is true, the other if it is false. Guarantees one path always executes.

Syntax:

if (condition) {
    // true block
} else {
    // false block
}
nested if
Multiple Levels

Theory: An if inside another if. Used when decisions depend on previous decisions (e.g., login → role check → action).

Syntax:

if (condition1) {
    if (condition2) {
        // both true
    }
}
switch
Multi-way

Theory: Replaces long if-else chains when comparing one variable against many fixed values (like menu choices).

Syntax:

switch (expression) {
    case value1:
        // ...
        break;
    case value2:
        // ...
        break;
    default:
        // none matched
}

Important: Without break;, execution "falls through" to the next case.

3. Syntax — Comparison Table

StatementBest ForTests
ifSingle conditionOne true/false
if-elseTwo-way branchingTrue vs false
nested ifDependent conditionsMany levels deep
switchMany fixed valuesEquality with constants

4. Example

#include <stdio.h>

int main() {
    int marks = 78;

    // if-else
    if (marks >= 35) {
        printf("Result: PASS\n");
    } else {
        printf("Result: FAIL\n");
    }

    // nested if
    if (marks >= 35) {
        if (marks >= 75) {
            printf("Grade: A\n");
        } else {
            printf("Grade: B\n");
        }
    }

    // switch
    int choice = 2;
    switch (choice) {
        case 1:
            printf("You chose Tea\n");
            break;
        case 2:
            printf("You chose Coffee\n");
            break;
        case 3:
            printf("You chose Milk\n");
            break;
        default:
            printf("Invalid choice\n");
    }

    return 0;
}

5. Output

Result: PASS
Grade: A
You chose Coffee

6. Common Mistakes

❌ Using = instead of == in conditions: if (x = 5) assigns, doesn't compare.
❌ Forgetting braces { } — only the very next statement belongs to the if.
❌ Putting ; right after if (cond); — the ; ends the statement and the block runs unconditionally.
❌ Forgetting break in switch cases — execution falls through to the next case.
❌ Using switch with float or string — only int and char are allowed.

1. Introduction

Loops let you repeat a block of code multiple times without writing it again. Printing 1 to 1000 by hand would be 1000 printf statements — a loop does it in 3 lines. Loops are at the heart of almost every program.

2. Theory & Syntax

for loop
Counted

Theory: Best when you know in advance how many times to repeat. Has 3 parts in one line: initialize, condition, update.

Syntax:

for (initialization; condition; update) {
    // body
}

Flow: init → check condition → run body → update → repeat.

while loop
Condition-based

Theory: Best when you don't know the count, only the condition. Tests the condition before each iteration.

Syntax:

while (condition) {
    // body
}

Flow: check condition → run body → repeat. If false at start, body never runs.

do-while loop
Run-at-least-once

Theory: Like while, but the body runs at least once before checking the condition.

Syntax:

do {
    // body
} while (condition);

Flow: run body → check condition → repeat. Note the ; after while.

break
Exit Loop

Theory: Immediately exits the current loop, regardless of the condition. Useful when you found what you were looking for.

Syntax:

for (...) {
    if (something) break;
}
continue
Skip Iteration

Theory: Skips the rest of the current iteration and jumps to the next one. Loop does not stop — just this round.

Syntax:

for (...) {
    if (skipThis) continue;
    // code below is skipped this round
}

3. Syntax — Comparison Table

LoopWhen To UseRuns Body WhenMin. Runs
forKnown countCondition true0 times
whileUnknown countCondition true0 times
do-whileMust run onceCondition true (after body)1 time
breakEarly exit needed——
continueSkip current round——

4. Example

#include <stdio.h>

int main() {
    // for — print 1 to 5
    printf("for loop:\n");
    for (int i = 1; i <= 5; i++) {
        printf("%d ", i);
    }
    printf("\n");

    // while — countdown
    printf("while loop:\n");
    int n = 3;
    while (n > 0) {
        printf("%d ", n);
        n--;
    }
    printf("\n");

    // do-while — at least once
    printf("do-while loop:\n");
    int x = 0;
    do {
        printf("%d ", x);
        x++;
    } while (x < 3);
    printf("\n");

    // break — stop early
    printf("break:\n");
    for (int i = 1; i <= 10; i++) {
        if (i == 6) break;
        printf("%d ", i);
    }
    printf("\n");

    // continue — skip 3
    printf("continue (skip 3):\n");
    for (int i = 1; i <= 5; i++) {
        if (i == 3) continue;
        printf("%d ", i);
    }
    printf("\n");

    return 0;
}

5. Output

for loop:
1 2 3 4 5
while loop:
3 2 1
do-while loop:
0 1 2
break:
1 2 3 4 5
continue (skip 3):
1 2 4 5

6. Common Mistakes

❌ Missing the update in for: for (int i=0; i<5; ) — infinite loop.
❌ Writing while (i < 5); with a stray semicolon — body becomes empty, loop never ends.
❌ Forgetting to update the counter in while — same infinite-loop trap.
❌ Using = instead of == in loop conditions.
❌ Putting ; after while(...); in do-while outside the statement — actually this one is required; just don't miss it entirely.
❌ Confusing break (exits loop) with continue (skips round). Both useful, different jobs.

  3. Print the multiplication table of a number entered by the user.
  4. Find the factorial of a number using a loop.
  5. Print all even numbers between 1 and 50 using continue to skip odd ones.
  6. Stop a loop using break as soon as the user enters a negative number.
  7. Predict the output of a do-while loop when the condition is false from the start. (Hint: runs once.)
